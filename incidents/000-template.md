# Incident Report Template

Copy this file for every new incident. Keep the structure consistent — recruiters and hiring managers skim these, so predictable formatting matters more than creative formatting.

---

## INC-XXX — [Short, specific title]

**Date:** YYYY-MM-DD HH:MM (timezone)
**Severity:** Low / Medium / High / Critical
**Status:** Resolved / Workaround / Ongoing
**Component(s) affected:** [e.g. Wazuh agent, Netplan, CrowdSec]

### Summary
One or two sentences. What happened, in plain language.

### Symptom
What you actually saw — exact error text, command output, screenshot reference.

```
paste exact error/output here
```

### Diagnosis
What you checked, in order, and what you found. Show the troubleshooting process, not just the answer — this is what proves analytical thinking.

### Root Cause
The actual underlying reason, stated in one or two sentences.

### Resolution
Exact commands or config changes that fixed it.

```bash
command here
```

### MITRE ATT&CK / Security+ Mapping (if applicable)
- Technique: TXXXX — Name
- Security+ Domain: D# — Domain name

### Lessons Learned
What you'd do differently, or what this taught you that applies beyond this one incident.
